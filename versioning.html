peter pan

Peter Pan is a RESTful API for Comcast to create sales orders with MDSi.

Versioning

We use flavor semantic versioning for the API.

Instead of Major.Minor.Patch, however, we use Major.Minor.Build.Revision. The Major and Minor numbers are part of the URL.

Major

If we make a breaking change to the API, we'll update the major version. Updating a major version may require code changes on your part.

Minor

If we add new features but don't change any contracts, we'll update the minor version. You can safely upgrade minor versions without worrying about your code breaking.

Build

This is the internal build number in YYDoY (Year + Day of Year) format. For example, 1.0.19323.1 was deployed on the 323rd day of 2019, which is November 19, 2019.

Revision

This is the number of times the code was built during the day. 1.0.19323.7 would indicate 7 revisions on November 19th.

Examples

Major and minor version changes have their own URLs. Changes to build and revision do not change the URL.
  • The base URL of v1.0 of the API would be https://mdsi-peter-pan-eus-uat-v1-0.azurewebsites.net/api
  • The base URL of v1.1 of the API would be https://mdsi-peter-pan-eus-uat-v1-1.azurewebsites.net/api
  • The base URL of v2.0 of the API would be https://mdsi-peter-pan-eus-uat-v2-0.azurewebsites.net/api
